# Session Summary — 2026-04-14

## Project Overview

**Goal:** Uncertainty-aware left-arm controller using MDM (Motion Diffusion Model) + MPC.
The pipeline is:
1. Record videos of a person performing arm motions while seated.
2. Extract frames → run SAM 3D Body (MHR) pose estimator → convert SMPL positions to
   HML263 features → save as MDM-compatible dataset.
3. Fine-tune MDM on the dataset.
4. At inference, fix the body (non-arm features) via MDM inpainting and generate
   left-arm motion conditioned on an initial pose.

## HML263 Feature Layout (263 dims)

```
[0:1]     root angular velocity (heading)
[1:3]     root linear velocity XZ
[3:4]     root height Y above ground
[4:67]    local joint positions — joint j (1-based, joints 1–21):
              [4+(j-1)*3 : 4+(j-1)*3+3]
          IMPORTANT: Y is ABSOLUTE height above ground, not root-relative.
          Only XZ are root-relative (root is at XZ=0 in local frame).
[67:193]  6D rotations — joint j (1-based): [67+(j-1)*6 : 67+(j-1)*6+6]
[193:259] joint velocities — joint j (0-based): [193+j*3 : 193+j*3+3]
[259:263] foot contact flags
```

`recover_from_ric` (MDM) adds only root XZ to decoded joint positions,
confirming the absolute-Y convention.

**SMPL-22 left arm joints (0-indexed):** 16 (left_shoulder), 18 (left_elbow), 20 (left_wrist)

## Critical Bugs Fixed (Previous Session)

### 1. HML263 encoding bug (`smpl_to_hml263.py`)
Local joint Y was stored as root-relative instead of absolute height above ground.
`recover_from_ric` only adds root XZ, so decoded joints had wrong heights.

**Fix:** Compute `ground_y = float(np.min(positions[:, :, 1]))` before building
local_y, then use `local_y = positions[:, :, 1] - ground_y` (absolute).

### 2. Double vertical translation in decode (`visualize_sitting_pose.py`)
`rot2xyz(vertstrans=True)` after `recover_from_ric` caused double root-Y offset.

**Fix:** Skip `rot2xyz` entirely for `hml_vec` data; use `recover_from_ric` output directly:
```python
ric = recover_from_ric(unnorm, 22)  # (1, 1, 1, 22, 3)
return ric[0, 0, 0].cpu().numpy()   # (22, 3)
```

### 3. MDM empty dataset assertion
`test.txt` / `val.txt` had only 1 unique ID; MDM asserts `len(dataset) > 1`.

**Fix:** Always pad splits to ≥ 2 IDs (repeating train IDs is fine).

### 4. CUDA compatibility (Blackwell GPU)
Machine has RTX PRO 6000 Blackwell (sm_120). cu121 PyTorch doesn't support it.

**Fix in `pyproject.toml`:**
```toml
[[tool.uv.index]]
name = "pytorch-cu128"
url = "https://download.pytorch.org/whl/cu128"
torch = { index = "pytorch-cu128" }   # >=2.7.0
```

### 5. `show_conversion.py` — wrong HML263 decode
`_hml263_to_local_positions` was adding root_y to already-absolute Y.
**Fix:** `positions[:, 1:] = rel_pos` (not `+= root_pos`).

## Inference Setup (`sample_leftarm.py`)

MDM inpainting is **non-causal** (full-sequence generation at each denoising step).
The inpainting mask hard-replaces masked features at every step.

**Current config:**
```python
N_PREFIX = 10        # lock first 10 frames to initial pose
FIX_BODY_ALL_FRAMES = False   # body is free; only prefix is locked
```

With `FIX_BODY_ALL_FRAMES = True`, a `NOT_L_ARM_MASK` is applied to all frames.

**Why N_PREFIX=10:** With only 1 frame pinned and a pretrained MDM prior (mostly
standing data), the body jumped from sitting to standing at frame 2. Locking 10 prefix
frames prevents this.

## Dataset Builder: `build_mdm_dataset.py`

### Key features added this session

#### `--fix_body` flag
Locks all non-arm HML263 features to their frame-0 values across every frame.
Makes fixed-body inference in-distribution during training.

```python
_L_ARM_JOINTS = {16, 18, 20}  # left_shoulder, left_elbow, left_wrist

def _arm_feature_mask() -> np.ndarray:
    mask = np.zeros(263, dtype=bool)
    for j in _L_ARM_JOINTS:
        mask[4 + (j-1)*3 : 4 + (j-1)*3 + 3] = True   # positions
        mask[67 + (j-1)*6 : 67 + (j-1)*6 + 6] = True  # rotations
        mask[193 + j*3 : 193 + j*3 + 3] = True         # velocities
    return mask

def _lock_body_to_frame0(hml263):
    arm_mask = _arm_feature_mask()
    result = hml263.copy()
    result[:, ~arm_mask] = result[0:1, ~arm_mask]
    return result
```

#### `--n_augment N` + `--noise_std FLOAT` flags
Save N additional noisy copies of each trajectory (default: 0).
Noise is applied only to arm features (not body), in normalized HML263 space.

```python
aug_rng = np.random.default_rng(seed + motion_id)
for _ in range(n_augment):
    hml263_aug = hml263.copy()
    noise = aug_rng.standard_normal(hml263.shape).astype(np.float32)
    hml263_aug[:, arm_mask] += noise[:, arm_mask] * noise_std
    # save hml263_aug with its own ID and same caption
```

## MDM Training Command

```bash
cd src/uncertain_feedback/motion_generators/mdm/motion-diffusion-model
python -m train.train_mdm \
    --save_dir save/my_model \
    --dataset humanml \
    --data_dir /path/to/my_mdm_dataset/ \
    --num_steps 10000 \
    --batch_size 8 \
    --lr 1e-4
```

**Cache gotcha:** MDM caches the dataset in `dataset/HumanML3D/`. If you regenerate
the dataset, delete the cache (`*.db` or pickle files) or the loader will use stale data.

## Dataset Format (MDM-compatible HumanML3D)

```
output_dir/
├── new_joint_vecs/000001.npy   # (N, 263) float32 normalized HML263
├── texts/000001.txt            # caption#w/POS w/POS ...#0.0#0.0
├── train.txt                   # one ID per line (zero-padded 6-digit)
├── val.txt                     # ≥ 2 IDs required (pad from train if needed)
├── test.txt                    # ≥ 2 IDs required
├── Mean.npy                    # (263,) — copied from HumanML3D
└── Std.npy                     # (263,) — copied from HumanML3D
```

Text file format: `caption#word/POS word/POS ...#0.0#0.0`
POS tags: Universal POS via spaCy `en_core_web_sm`.
